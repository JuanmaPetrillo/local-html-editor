## Current status

Project phase: active development - browser-first static prototype.

Current milestone: Phase 9E complete (V2 polish — copy/paste, z-order, italic/underline/opacity, snap to grid, better layers).

## Latest summary

The primary app is now **V2** (`apps/desktop-v2/`), a single-file direct-canvas editor. Implementation is through Phase 10 (usability hardening). V1 (`apps/desktop/`) remains available as a reference but is not the current pilot target.

V2 implemented capabilities: dual-iframe architecture (edit iframe: allow-same-origin; preview iframe: allow-scripts), overlay selection engine, transparent edit-overlay capturing all pointer events, overlay drag/resize, inspector (X/Y/W/H, color, font, size, bold/italic/underline/opacity, alignment, font family, image replace), multi-slide navigation, slide add/delete/duplicate, copy/paste (Ctrl+C/V), z-order (bring-front/send-back), snap-to-grid, undo/redo (Ctrl+Z/Y), explicit Save Project / Open Project (`.lheproj-v2.json`), export sanitized HTML, Preview/Edit mode split with full sandboxing.

ZIP remains preflight-only (no extraction/listing/export). Local image replacement is implemented for selected safe `<img>` objects. Persistence/autosave and packaging/framework conversion remain unimplemented.

Current V2 app files:

- `apps/desktop-v2/index.html`
- `apps/desktop-v2/src/app-v2.mjs`

Build output: `dist/index.html` + `dist/app-v2.bundle.js` (via `npm run build`).
Pilot package: `pilot-dist/local-html-editor-pilot-<timestamp>/` (via `npm run package:pilot`).

## Open risks

- HTML generated by Copilot may vary substantially; importer is best-effort.
- Arbitrary JS in imported decks is preserved in preview but stripped in edit/export mode.
- Browser security policies may vary on enterprise-managed browsers.
- `convertToAbsolute` called during drag/nudge on normal-flow elements can shift surrounding CSS flex/grid layouts; Ctrl+Z to undo.
