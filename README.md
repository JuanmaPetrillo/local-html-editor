# Local HTML Presentation Editor

A fully local editor for HTML presentations, especially Microsoft 365 Copilot-generated presentations.

The long-term product goal is simple: a non-technical user should be able to open a downloaded HTML presentation, edit visible text, replace images, move/resize elements with drag-and-drop, preview the result, and export a working HTML file again without sending sensitive content to any third party.

## Product principles

1. **Full local execution**: presentation files, drafts, assets, logs, and exports stay on the user's machine.
2. **No admin requirement for end users**: the distributed app must run/install under a normal Windows user profile.
3. **No-code editing**: the primary experience must feel like a presentation editor, not an IDE.
4. **Safe by default**: imported HTML is untrusted. It must never gain access to privileged app APIs or broad filesystem access.
5. **Preserve first, edit second**: when an imported component cannot be safely edited, preserve and preview it rather than rewriting it destructively.
6. **Small Codex tasks**: use repository docs as durable context and run Codex in PR-sized milestones.

## Recommended architecture

Default stack:

- Desktop shell: Tauri 2
- Frontend: React + TypeScript + Vite
- Visual editor foundation: GrapesJS for page-builder/editor state, with a simplified custom UI
- Precision movement: Moveable or interact.js for drag, resize, snap, and nudge interactions
- Import/export: parse5 for HTML parsing and JSZip for ZIP bundles
- Security: sandboxed preview, strict Content Security Policy, no remote network by default, no telemetry
- Testing: Vitest, Playwright, security fixtures, accessibility checks, import/export roundtrip fixtures
- Distribution: Windows user-scope installer or portable bundle generated by GitHub Actions

## Strategy (current and future)

The long-term product is a local visual editor, while today's implementation is a browser-first MVP foundation/proof of concept.

Plain-language scope statement:

> The goal is not to make every possible HTML file fully editable. The goal is to make common presentation micro-edits possible and preserve/lock complex content.

Current strategy summary:

- Support arbitrary HTML imports as best-effort.
- Classify content into editable, partially editable, or locked/preserved.
- Support edit-ready Copilot metadata when available, but do not require it.
- Preserve unsupported complex components instead of destructively rewriting them.
- Use prompt training + temporary power-user helper workflows for immediate value while the visual-editor UX matures.

## Current implementation state

Active development is currently a **browser-first static prototype**.

### MVP readiness (current browser-first implementation)

#### Current capabilities

- Open local `.html`, `.htm`, and `.zip` files from the shell.
- Scan selected imports and show local safety/reporting summaries.
- Show a sanitized safe preview for HTML/HTM in a sandboxed iframe (`sandbox=""`, `referrerpolicy="no-referrer"`).
- Discover editable text candidates from HTML/HTM with source-span tracking and entity decode/encode roundtrip.
- Discover visual objects from HTML/HTM with editability classification (editable / partially-editable / locked-preserved) and plain-language confidence reasons.
- Render a trusted-shell overlay of positioned boxes over the safe preview stage for geometry-ready objects.
- Select a visual object and bridge to the text-edit workflow via exact source-span matching.
- Draft and apply multiple in-memory text replacements to a working preview copy (overlap detection, descending-offset application).
- Move selected overlay-ready inline-style objects using trusted-shell nudge buttons with cumulative in-memory move patch state.
- Reset the working preview back to the original imported content.
- Export a user-initiated edited HTML copy locally (download), without mutating the original import, including movement patches when present.

#### Current limitations

- ZIP is preflight-only (no ZIP extraction/listing/export).
- No image replacement.
- Button-based nudge movement only; no free drag or resize handles yet (see ROADMAP.md).
- No project persistence/autosave/reopen flow.
- No iframe internals access (`contentDocument` / `contentWindow`) and no `postMessage` bridge.
- No Tauri/React/Vite/TypeScript conversion in this implementation branch.
- Overlay positions are derived from inline pixel geometry in the HTML source; elements without explicit inline `left`/`top`/`width`/`height` pixel styles will not have overlay boxes.

#### How to run locally

From repository root:

```bash
npm ci
npm run build
```

Then open `dist/index.html` in a local browser.

#### How to test the MVP flow

1. Open an `.html` or `.htm` file with **Open HTML/ZIP**.
2. Review file scan/status and safe preview summary.
3. Choose an editable text candidate.
4. Enter draft replacement text.
5. Apply draft to preview (in-memory only).
6. Export edited HTML copy (local download).

#### Implementation ownership notes

- Exported HTML is built from the original imported HTML text plus in-memory text patches (not from sanitized preview `srcdoc`).
- No persistence/autosave exists in this slice.
- No image replacement or drag/resize editing exists in this slice.


### Manual smoke checklist (current browser-first MVP)

1. Open an `.html` or `.htm` file with **Open HTML/ZIP**.
2. Confirm scan summary, import report, and import manifest appear.
3. Confirm safe preview appears.
4. Select a text candidate.
5. Type replacement text.
6. Apply draft to preview.
7. Apply a second candidate patch.
8. Reset preview to original.
9. Apply a patch again.
10. Export edited HTML.
11. Open exported HTML locally.
12. Confirm the originally imported source file is unchanged.
13. Open a `.zip` file and confirm ZIP remains preflight-only.
14. Open an unsupported file and confirm editing/export are unavailable.


- `apps/desktop/src/importer.mjs` owns selected-file reads (`.text()` / `.arrayBuffer()`) and import status construction.
- `apps/desktop/src/exporter.mjs` is a pure helper module for edited HTML export serialization logic.
- Edited HTML export is user-initiated and local-only via browser download behavior.

## Non-goals for the MVP

- No cloud sync.
- No collaboration features.
- No user accounts.
- No telemetry.
- No AI calls inside the editor.
- No arbitrary website builder scope.
- No promise to safely edit every possible HTML/JS web app.

## Future target MVP definition (not fully implemented in the current browser-first slice)

The MVP is successful when a normal user can:

1. Open an HTML file or ZIP bundle generated by Microsoft 365 Copilot Chat.
2. Edit visible text.
3. Replace images and alt text.
4. Move and resize selected elements with drag handles and keyboard nudging.
5. Preview safely with scripts off by default.
6. Export a working HTML file or ZIP bundle.
7. Do all of the above offline and without administrator rights.
